
# Epidura: Staged Remote IPC Process Injection Tool

Project **Epidura** is a rust-based module that performs **Staged Remote Command Injection** and **IPC Hijacking**. By separating the payload delivery from the execution trigger, this tool effectively bypasses traditional behavioral heuristics that rely on the suspicious triad of immediate memory allocation and execution.

**⚠️ Please Note**: This project is strictly for **Educational and Authorized Penetration Testing**. I am not responsible for any of the shenanigans you guys pull.

---

## 🛡️ Core Concept: Breaking the Injection Chain

Traditional EDR solutions hunt for a specific linear sequence: `VirtualAllocEx` → `WriteProcessMemory` → `CreateRemoteThread`. Epidura breaks this chain by having a **Dormant Stager**.

*   **Asynchronous Execution**: The injector creates a kernel-level **Named Pipe** server and enters a `Wait` state. It performs no malicious actions upon startup, making it invisible to most automated sandboxes.
*   **External Orchestration**: Execution is only triggered when an external process (the trigger) connects to the pipe and sends a control byte.
*   **API Hijacking**: Instead of noisy shellcode, the tool demonstrates the ability to hijack a target process's UI and internal state through existing Windows API exports.

## 🛠️ Technical Walkthrough

### 1. Stager (Rust Infrastructure)
The Rust binary acts as the listener. It establishes a duplex communication channel in the Windows Kernel using the `windows` crate. You can rename the pipe to something inconspicuous (e.g., `\\.\pipe\spooler_svc`) to blend in with the hundreds of legitimate pipes. For this project, I have named it to `SpoolerService`.

### 2. Trigger (PowerShell Client)
The remote command is sent via a lightweight PowerShell script. This simulates a post-exploitation scenario where an attacker interacts with a pre-placed stager to execute commands without dropping new files to disk.

### 3. Impact for this Project (UI Manipulation)
Upon receiving the trigger, Epidura locates the target process's window handle (`HWND`) and executes a **Renaming Hook**. I have kept the impact harmless, but it serves as a functional proof-of-concept for **Exported Function Hijacking** and **Integrity Violation**. Obviously, you can tweak it according to your engagement's requirements.

### 💥 Impact & Red Team Application

This staged IPC-driven model provides several high-value advantages for advanced red teaming and adversarial simulation.

---

*   **Advanced EDR & Behavioral Evasion**: Traditional shellcode injection relies on the creation of high-entropy memory regions often marked as `PAGE_EXECUTE_READWRITE` which are primary triggers for modern EDR telemetry. By utilizing **API Orchestration**, Epidura leverages legitimate signed system exports (like `SetWindowTextW`). This allows the attacker to manipulate the target process using its own trusted logic, significantly **reducing the noise** generated by unverified code execution and bypassing memory-scanning heuristics.

*   **Low-Profile Persistence via IPC Backdoors**: The Named Pipe server is a stealthy low-profile persistence mechanism. Unlike standard beacons that generate detectable traffic, this stager remains entirely passive within the kernel's IPC subsystem. It acts as a **Listen-Only** backdoor, remaining dormant and showing zero suspicious behavior until the attacker provides a trigger byte via the pipe. This effectively evades sandboxes that rely on time-outs or immediate behavioral triggers.

*   **Integrity Violation PoC**: The project is a functional demonstration of how an attacker can breach the integrity of a protected process without ever needing to execute a single line of traditional shellcode. **Process A** (Epidura) was able to reach into the memory space of **Process B** (Notepad) and modify its metadata without permission. This is an Integrity Violation. If the OS allows it to change the title bar of a system app, it shows that the `AppContainer` or process isolation boundary has a hole that could potentially be used for more dangerous memory operations.

## 💻 Execution Guide

- **Build**
```bash
cargo build --release
```
- **Deploy**
```bash
.\target\release\epidura.exe <PID> (target any open notepad).
```
- **Trigger** (in another PowerShell window)
```powershell
$pipe = [System.IO.Pipes.NamedPipeClientStream]::new(".", "SpoolerService", [System.IO.Pipes.PipeDirection]::Out)
$pipe.Connect(); $pipe.WriteByte(0x01); $pipe.Close()
```
